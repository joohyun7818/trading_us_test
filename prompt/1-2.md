

당신은 Python/Pydantic 전문 개발자입니다.

## 배경

`api/services/rag_analyzer.py`의 `analyze_stock()` 함수는 Ollama LLM(qwen3:8b)의 JSON 응답을 파싱하여
sentiment_score, confidence, key_issues, is_priced_in, outlook, rationale 필드를 반환합니다.

문제: LLM이 예상 범위 밖의 값(예: sentiment_score=85.0, confidence=200)을 반환할 수 있으며,
이 값이 검증 없이 `trading_engine.py`의 `text_score = ((text_sentiment + 1) / 2) * 100` 계산에
전파되어 비정상적 시그널을 생성할 수 있습니다.

## 작업 요구사항

1. `api/services/models/` 디렉토리에 `analysis_models.py` 파일을 생성하세요.
2. 아래 Pydantic 모델을 정의하세요:

```python
class RAGAnalysisResult(BaseModel):
    sentiment_score: float = Field(default=0.0, ge=-1.0, le=1.0)
    confidence: float = Field(default=0.0, ge=0.0, le=1.0)
    key_issues: list[str] = Field(default_factory=list, max_length=10)
    is_priced_in: bool = Field(default=False)
    outlook: Literal["bullish", "bearish", "neutral"] = Field(default="neutral")
    rationale: str = Field(default="", max_length=500)
```

1. `rag_analyzer.py`의 `analyze_stock()` 함수에서 `_parse_json_response()` 결과를 `RAGAnalysisResult`로 검증하세요. 검증 실패(ValidationError) 시 `_DEFAULT_RESULT`를 반환하세요.
2. 검증 실패 시 `logger.warning()`으로 실패 원인과 원본 값을 기록하세요.
3. `analyze_macro()` 함수에도 유사한 `MacroAnalysisResult` 모델을 적용하세요.

## 제약

* 기존 `_parse_json_response()` 함수는 수정하지 마세요 (폴백 파싱 유지).
* `_DEFAULT_RESULT`의 값과 모델 기본값을 일치시키세요.
* 반환 타입은 여전히 dict로 유지하세요 (`.model_dump()` 사용).

## 출력

1. `api/services/models/analysis_models.py` 전체
2. 수정된 `api/services/rag_analyzer.py` 전체

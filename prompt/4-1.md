
AlphaFlow US에 ATR 기반 변동성 비례 포지션 사이징을 구현해라.

■ 배경
현재 모든 거래가 max_order_amount=$1,000 고정이다.
고변동성 종목(ALB, COIN)에서 -8% SL이 빈발하고,
저변동성 종목(JNJ, PG)은 +15% TP 도달에 수개월 걸린다.
종목별 리스크 노출을 균등화해야 한다.

■ 할 일

1) api/services/position_sizer.py 신규 생성

   - async def calculate_position_size(symbol, signal_score, account_equity,
     current_positions, max_positions) -> dict
   - 공식:
     risk_amount = account_equity × risk_pct (settings에서 읽기, 기본 0.01)
     dollar_risk = atr_14 × hard_stop_atr_mult (4-2에서 만든 설정값 사용)
     shares = risk_amount / dollar_risk
   - signal_score 스케일링:55-64 → ×0.7, 65-79 → ×1.0, 80-89 → ×1.2, 90+ → ×1.5
     -제약 적용:
     · 최대 단일 주문:account_equity × 5%
     · 최소 주문:$200
     · 동일 섹터 합계:account_equity × 30% 이하
     (stocks 테이블에서 sector_id 조회, current_positions에서 섹터별 합산)
   - 반환:{symbol, order_amount, quantity, risk_per_trade, atr_14, sizing_reason}
     -atr_14는 stocks 테이블의 atr 컬럼에서 조회, 없으면 latest_price × 0.02
2) api/services/auto_trader.py 수정

   - BUY 주문 시 기존 max_order_amount 고정 대신 calculate_position_size 호출
   - sizing 결과의 order_amount 사용
   - order_amount <= 0이면 건너뛰기 (섹터 상한 등)
3) api/services/backtester.py 수정

   - BacktestConfig에 추가:
     use_atr_sizing:bool = False
     risk_per_trade_pct:float = 1.0
     max_single_order_pct:float = 5.0
     sector_cap_pct:float = 30.0
     -use_atr_sizing=True일 때:
     · 매수 시 notional을 atr 기반으로 계산 (stock_daily에서 해당 종목 해당 날짜의 atr_14 사용)
     · 섹터 상한 체크 (stock_daily에 sector 정보 없으면 무시하고 넘어가도 됨)
     -use_atr_sizing=False이면 기존 max_order_amount 로직 유지

■ 제약
-4-2의 exit_manager와 독립적으로 동작해야 함 (둘 다 켜거나 하나만 켤 수 있음)
-코딩 규칙 준수

■ 완료 기준
-백테스트에서 use_atr_sizing=true 실행 시 종목별 order_amount가 다양하게 나타남
-고변동성 종목의 주문 금액이 저변동성 대비 작음
-use_atr_sizing=false이면 기존 결과와 동일

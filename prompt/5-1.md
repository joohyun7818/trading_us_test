
AlphaFlow US에 시스템 모니터링과 Slack 알림을 추가해라.

■ 할 일

1) DB 마이그레이션

   - system_alerts 테이블: id SERIAL PK, severity VARCHAR(10), category VARCHAR(30),
     message TEXT, auto_action VARCHAR(30), resolved BOOLEANDEFAULTFALSE,
     created_at TIMESTAMPTZDEFAULT NOW()
   - settings 초기값: slack_webhook_url = '' (빈 문자열)
2) api/services/monitoring.py 신규 생성

   - async def run_health_checks() -> list[dict]
   - 점검 항목:
     a) stocks.updated_at이 2시간 이상 지연 → WARNING "data_freshness"
     b) 최근 1시간 news_articles 신규 < 10건 → WARNING "news_collection"
     c) Ollama health_check 실패 → CRITICAL "ollama_down"
     d) 24시간 signals 건수 = 0 → WARNING "no_signals"
     e) 1시간 내 BUY 시그널 > 50건 → WARNING "excessive_buy"
     f) 최근 1시간 trades에서 status='failed' ≥ 3건 → CRITICAL "order_failures"
   - 각 항목: {severity, category, message, auto_action(옵션)}
3) api/services/alerting.py 신규 생성

   - async def process_alerts(alerts: list[dict])
   - 모든 알림 → system_alerts 테이블 INSERT
   - severity == "CRITICAL" → Slack 전송 + auto_action 실행
     · auto_action == "pause_trading" → settings의 auto_trade_enabled를 "false"로 UPDATE
   - Slack 전송: settings의 slack_webhook_url로 httpx POST
     · URL 비어있으면 스킵 (경고 로그만)
   - async def send_daily_report()
   - 당일 거래 건수, BUY/SELL 수, 총 P&L, 알림 건수를 집계하여 Slack 전송
4) api/services/scheduler.py 수정

   - 5분 간격 job 추가: run_health_checks → process_alerts
   - 매일 16:30 EST job 추가: send_daily_report

■ 완료 기준

- Ollama를 끄면 CRITICAL 알림 발생하고 system_alerts에 기록됨
- slack_webhook_url 설정 시 Slack 메시지 수신
- auto_trade_enabled가 자동으로 "false"로 변경됨

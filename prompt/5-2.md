
AlphaFlow US에 실전 전환 가능 여부를 자동 판단하는 게이트를 구현해라.

■ 할 일

1) api/services/live_gate.py 신규 생성

   - GATE_CRITERIA 딕셔너리 정의:
     min_paper_weeks:12
     min_sharpe:0.8
     max_mdd_pct:-15.0
     min_win_rate_pct:45.0
     min_profit_factor:1.2
     min_total_trades:50
     max_critical_alerts_30d:0
     dynamic_exit_enabled: True
     atr_sizing_enabled: True
   - async def evaluate_live_readiness() -> dict
   - daily_snapshot 테이블에서 최근 12주 데이터 조회
     (daily_snapshot이 없으면 backtest 결과로 대체 가능하도록 폴백)
   - 각 기준별 required vs actual 비교
   - settings 테이블에서 dynamic_exit_enabled, atr_sizing_enabled 값 확인
   - system_alerts에서 최근 30일 CRITICAL 건수 확인
   - 반환:
     {
     ready: bool (모든 기준 통과),
     score: 통과 비율 (0-100),
     criteria: { 기준명: {required, actual, passed} },
     blocking: 미통과 항목 리스트,
     staged_plan: {
     stage1: {capital_pct:1, weeks:4, gate:"Sharpe≥0.5"},
     stage2: {capital_pct:3, weeks:4, gate:"Sharpe≥0.8"},
     stage3: {capital_pct:5, weeks:4, gate:"monthly alpha>0"}
     }
     }
2) api/routers/system.py 또는 별도 라우터에 엔드포인트 추가

   - GET /api/system/live-readiness → evaluate_live_readiness

■ 완료 기준

- 현재 상태에서 호출 시 ready=False, blocking에 최소 3개 항목

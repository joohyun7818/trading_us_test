당신은 FastAPI 보안 전문 개발자입니다.

## 배경

현재 `api/main.py`에는 인증/인가 미들웨어가 없습니다. 누구든 `/api/batch/run`, `/api/alpaca/order/buy`, `/api/alpaca/order/sell` 등을 호출하면 주문이 실행됩니다. CORS도 `allow_origins=["*"]`로 전면 개방되어 있습니다.

## 작업 요구사항

1. `api/core/auth.py` 파일을 생성하세요:
   * `API_KEY` 환경변수(.env)에서 API 키를 로드합니다.
   * FastAPI `Depends`로 사용할 `verify_api_key()` 의존성 함수를 구현합니다.
   * `X-API-Key` 헤더를 검사하고, 불일치 시 `HTTPException(401)`을 반환합니다.
2. 아래 라우터에 인증을 적용하세요 (읽기 전용 API는 제외):
   * `/api/batch/run` (POST)
   * `/api/alpaca/order/*` (POST)
   * `/api/alpaca/cancel/*` (POST)
   * `/api/rag/index` (POST)
   * `/api/rag/analysis/*` (POST)
   * `/api/macro/regime/calculate` (POST)
   * `/api/macro/settings/*` (PUT)
3. `api/core/config.py`의 `Settings`에 `API_KEY: str = ""` 필드를 추가하세요.
4. CORS `allow_origins`를 환경변수 `CORS_ORIGINS`(쉼표 구분)로 변경하세요. 기본값은 `["http://localhost:5173", "http://localhost:3000"]`으로 설정하세요.
5. `.env.example`에 `API_KEY`와 `CORS_ORIGINS` 예시를 추가하세요.

## 제약

* GET 엔드포인트(대시보드 조회, 상태 확인 등)에는 인증을 적용하지 마세요.
* 프론트엔드 SPA 서빙 경로에는 인증을 적용하지 마세요.
* API_KEY가 빈 문자열이면 인증을 비활성화하세요 (개발 편의).

## 출력

1. `api/core/auth.py` 전체
2. 수정된 `api/core/config.py` 전체
3. 수정된 `api/main.py` (CORS 변경 부분)
4. 인증 적용된 라우터 예시 1개 (alpaca.py)

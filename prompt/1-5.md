당신은 Python pytest 전문 테스트 엔지니어입니다.

## 배경

AlphaFlow US 트레이딩 시스템에는 테스트 코드가 전혀 없습니다. 503개 종목에 대해 자동 주문을 실행하는 시스템에 회귀 테스트가 없으면 코드 변경 시 시그널 왜곡을 사전에 감지할 수 없습니다.

## 작업 요구사항

### 1. 프로젝트 설정

* `pytest.ini` 또는 `pyproject.toml`에 pytest 설정을 추가하세요.
* `tests/` 디렉토리를 생성하고 `conftest.py`에 공통 fixture를 정의하세요.
* DB 의존성이 있는 함수는 `unittest.mock.AsyncMock`으로 모킹하세요.

### 2. tests/test_numeric_analyzer.py (10+ 테스트)

`api/services/numeric_analyzer.py`의 순수 함수들을 테스트합니다:

* `_score_rsi()`: RSI 20→95.0, RSI 50→50.0, RSI 80→20.0, RSI None→50.0
* `_score_macd()`: golden cross(macd > signal), dead cross, None 입력
* `_score_bollinger()`: pct_b=0→90, pct_b=0.5→50, pct_b=1.0→20, None→50
* `_score_volume()`: ratio=2.0→80, ratio=0.3→30
* `_score_52w_position()`: 52주 저점 근처→85, 52주 고점 근처→20
* `_score_atr()`: 적정 변동성(2~3.5%)→70, 과도 변동성(7%+)→25
* `calculate_numeric_score()`: DB 모킹 후 통합 테스트

### 3. tests/test_sentiment.py (8+ 테스트)

`api/services/sentiment.py`의 함수를 테스트합니다:

* `analyze_sentiment_keywords()`: 강한 긍정 텍스트→positive, 강한 부정→negative, 혼합→neutral
* 빈 텍스트→0.0/neutral
* "outstanding debt"에서 "outstanding"이 false positive가 되는지 확인
* priced-in 패턴 감지: "priced in", "already factored", "baked into"
* "old news" 패턴 감지 테스트

### 4. tests/test_trading_engine.py (8+ 테스트)

`api/services/trading_engine.py`의 함수를 테스트합니다:

* `_determine_signal()`: 70→BUY, 30→SELL, 50→HOLD, 경계값 70.0→BUY, 30.0→SELL
* `_apply_adjustments()`: priced_in -15, RSI≥75 -10, RSI≤25 +10, 조합 시 클램핑 [0, 100]
* 비전 패턴 조정: double_bottom confidence≥0.7 → +8, double_top → -8
* 조합 시나리오: base=72, priced_in → 72-15=57 → HOLD

### 5. tests/test_rag_analyzer.py (5+ 테스트)

* `_parse_json_response()`: 정상 JSON, ```json 래핑, trailing comma, 태그 제거
* 빈 응답 → 빈 dict
* 부분 필드만 있는 경우 regex 추출

## 제약

* 외부 API(Ollama, Alpaca, yfinance)는 모두 모킹하세요.
* DB 함수(`fetch_one`, `fetch_all`, `execute`)는 모두 AsyncMock으로 모킹하세요.
* 테스트 실행에 외부 서비스 의존이 없어야 합니다.
* `assert` 문에 설명 메시지를 포함하세요.

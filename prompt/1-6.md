당신은 Python asyncio 전문 개발자입니다.

## 배경

AlphaFlow US는 FastAPI 기반 fully async 아키텍처를 표방하지만, 다음 모듈에서 동기 I/O 호출이 async 함수 내에서 직접 실행되어 이벤트 루프를 블로킹합니다:

1. `news_crawler.py`의 `_crawl_finnhub()`: `finnhub.Client`는 동기 SDK
2. `macro_engine.py`의 `_calc_sp500_trend()`, `_calc_vix_score()`, `_calc_yield_curve()`, `_calc_market_rsi()`, `_calc_breadth()`: `yf.Ticker().history()`는 동기 호출
3. `sp500_loader.py`의 `_get_market_cap()`: `yf.Ticker().info`는 동기 호출
4. `price_crawler.py`의 `crawl_prices()`: `yf.download()`는 동기 호출
5. `chart_analyzer.py`의 `_generate_candlestick_chart()`: yfinance + mplfinance 동기
6. `alpaca_client.py`의 모든 함수: `TradingClient`, `StockHistoricalDataClient`는 동기 SDK

## 작업 요구사항

1. `api/core/utils.py`에 헬퍼 함수를 만드세요:

```python
Copyimport asyncio
from functools import partial
from typing import TypeVar, Callable, Any

T = TypeVar("T")

async def run_sync(func: Callable[..., T], *args: Any, **kwargs: Any) -> T:
    """동기 함수를 이벤트 루프 블로킹 없이 실행한다."""
    loop = asyncio.get_running_loop()
    return await loop.run_in_executor(None, partial(func, *args, **kwargs))
```

1. 위 6개 모듈의 동기 호출을 `await run_sync()`로 감싸세요. 예: `hist = await run_sync(spy.history, period="3mo")`
2. `alpaca_client.py`의 경우:
   * `get_account()`, `get_positions()` 등에서 SDK 동기 호출을 `run_sync()`로 감싸세요.
   * `submit_order()`도 마찬가지입니다.
3. `news_crawler.py`의 `_crawl_finnhub()`에서:
   * `finnhub.Client.company_news()`를 `run_sync()`로 감싸세요.

## 제약

* 함수 시그니처(async def)와 반환 타입은 변경하지 마세요.
* yfinance의 `download()`는 내부적으로 스레드를 사용하므로 `run_in_executor` 사용 시 thread pool 크기(max_workers)를 5로 제한하세요.
* 기존 에러 핸들링(try-except)을 유지하세요.

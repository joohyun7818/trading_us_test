
AlphaFlow US의 ChromaDB 검색을 bge-m3 + Gemini 임베딩 하이브리드로 통합해라.

■ 배경
현재 stock_news (bge-m3, Ollama)와 stock_news_gemini (Gemini) 두 컬렉션이 분리되어 있다.
검색 시 하나만 사용한다. 두 컬렉션을 결합하면 recall이 올라갈 수 있다.
단, 현재 text 가중치 = 0이므로 실전 효과는 text 축 채택 이후에 확인.

■ 할 일

1) api/services/hybrid_search.py 신규 생성

   - async def hybrid_search(query, symbol=None, top_k=5) -> list[dict]
   - 두 컬렉션에서 각각 top_k × 2 조회
     · bge:news_indexer의 컬렉션 + ollama_client.embed로 쿼리 임베딩
     · gemini:gemini_indexer의 컬렉션 + gemini_client.gemini_embed로 쿼리 임베딩
     -Reciprocal Rank Fusion으로 결합:
     rrf_score(doc) = Σ 1/(60 + rank_i) for each source
   - 결과를 rrf_score 내림차순 정렬, top_k 반환
   - 각 결과:{id, text, metadata, rrf_score, source}
     -한 컬렉션이 비어있거나 임베딩 실패 시 다른 쪽만 사용 (graceful fallback)
2) api/services/rag_engine.py 수정
   -settings 테이블에 rag_search_method 키 추가 (기본값 "bge")
   -"bge" → 기존 search_similar_news
   -"gemini" → gemini_indexer.search_gemini_news
   -"hybrid" → hybrid_search.hybrid_search
   -search_and_build_prompt에서 위 분기 적용

■ 완료 기준
-settings에서 rag_search_method="hybrid" 설정 후 RAG 분석 실행 시 에러 없이 작동
-두 컬렉션 모두 데이터가 있을 때 hybrid 결과가 단독보다 다양한 뉴스를 포함
-한 컬렉션이 비어있어도 에러 없이 나머지만으로 동작

당신은 PostgreSQL + Python 데이터 엔지니어입니다.

## 배경

AlphaFlow US 트레이딩 시스템에는 백테스트 프레임워크가 없습니다. 첫 단계로 과거 OHLCV + 기술지표 데이터를 저장할 전용 테이블이 필요합니다.

현재 `stocks` 테이블은 **최신 값만** 보유합니다 (current_price, rsi_14 등). 백테스트를 위해서는 날짜별 시계열 데이터가 필요합니다.

## 작업 요구사항

### 1. DB 마이그레이션 SQL (`scripts/migration_002_backtest.sql`)

```sql
CopyCREATE TABLE stock_daily (
    id BIGSERIAL PRIMARY KEY,
    symbol VARCHAR(10) NOT NULL,
    trade_date DATE NOT NULL,
    open NUMERIC(12,4),
    high NUMERIC(12,4),
    low NUMERIC(12,4),
    close NUMERIC(12,4) NOT NULL,
    volume BIGINT,
    adj_close NUMERIC(12,4),
    -- 사전 계산 기술지표
    rsi_14 NUMERIC(8,4),
    sma_20 NUMERIC(12,4),
    sma_60 NUMERIC(12,4),
    macd NUMERIC(12,6),
    macd_signal NUMERIC(12,6),
    macd_histogram NUMERIC(12,6),
    bollinger_upper NUMERIC(12,4),
    bollinger_lower NUMERIC(12,4),
    bollinger_pct_b NUMERIC(8,4),
    volume_ratio NUMERIC(8,4),
    atr_14 NUMERIC(12,4),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(symbol, trade_date)
);
CREATE INDEX idx_stock_daily_symbol_date ON stock_daily(symbol, trade_date);
CREATE INDEX idx_stock_daily_date ON stock_daily(trade_date);
```

### 2. 데이터 수집 스크립트 (`api/services/historical_loader.py`)

* S&P 500 전 종목의 최근 3년(약 756 거래일) OHLCV 데이터를 yfinance로 수집합니다.
* `price_crawler.py`의 기술지표 계산 함수들을 재사용하여 날짜별 지표를 계산합니다.
* 50개씩 배치로 다운로드하고, 진행률 로깅을 합니다.
* 이미 존재하는 (symbol, trade_date) 조합은 건너뜁니다 (ON CONFLICT DO NOTHING).
* 일별 증분 업데이트도 지원합니다 (가장 최근 trade_date 이후만 수집).

### 3. API 엔드포인트 (`api/routers/backtest.py`)

* `POST /api/backtest/load-history`: 히스토리컬 데이터 수집 트리거
* `GET /api/backtest/history-status`: 수집 상태 (종목 수, 날짜 범위, 누락 건수)

## 제약

* yfinance `download()`는 `run_sync()`로 감싸세요 (Phase 1-6 결과물 사용).
* 기술지표 계산 로직은 `price_crawler.py`의 함수를 import해서 재사용하세요. 단, 시계열 전체에 대해 계산하도록 수정이 필요하면 별도 함수로 분리하세요.
* `asyncpg.Pool`의 `executemany`를 활용하여 배치 INSERT 성능을 확보하세요.

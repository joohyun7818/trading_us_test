
AlphaFlow US에 백테스트 파라미터 그리드 탐색 기능을 추가해라.

■ 배경
4-2(동적 청산), 4-1(ATR 사이징) 적용 후 최적 파라미터 조합을 찾아야 한다.
hard_stop_atr_mult, trailing_stop_atr_mult, max_holding_days 등의 값을 수동으로 바꿔가며
백테스트를 반복하는 것은 비효율적이다.

■ 할 일

1) api/services/backtest_optimizer.py 신규 생성

   - async def run_sensitivity(base_config: dict, search_space: dict,
     objective: str ="sharpe", walk_forward: bool = False,
     train_ratio: float =0.7) -> dict
   - search_space 형태: {"param_name": [val1, val2, ...], ...}
   - 모든 조합을 itertools.product로 생성
   - 각 조합마다 backtester.run_backtest 호출, 결과에서 objective 지표 추출
     (objective는 "sharpe", "total_return", "calmar" 중 선택)
   - Sharpe 계산: daily_equity에서 일별 수익률 → (mean - rf) / std × sqrt(252)
   - walk_forward=True일 때:
     · 전체 기간의 trade_days를 train_ratio로 분할
     · train 구간에서 최적 조합 선택
     · test 구간에서 동일 조합 검증
     · overfit_gap = train_objective - test_objective
   - 반환:
     · ranked: 상위 20개 조합 + 각 조합의 objective 값
     · best:1위 조합
     · sensitivity: 파라미터별 { 값: 평균_objective } (어떤 파라미터가 가장 영향 큰지 파악용)
     · overfit_warning: overfit_gap >0.5인 조합이 상위 5개에 있는지
2) api/routers/backtest.py에 엔드포인트 추가

   - POST /api/backtest/sensitivity → run_sensitivity 호출

■ 주의

- 조합 수가 많으면 오래 걸린다. 진행률 로깅 필수 (매 10번째 조합마다 로그)
- BacktestConfig의 모든 필드를 search_space에 넣을 수 있어야 함

■ 완료 기준

- 다음 탐색이 정상 실행됨:
  search_space= {
  "hard_stop_atr_mult": [2.0, 2.5, 3.0],
  "trailing_stop_atr_mult": [1.5, 2.0, 2.5],
  "max_holding_days": [10, 15, 20]
  }
  (27조합)
- ranked 결과에서 best 조합의 Sharpe가 기본값(2.5/2.0/20) 대비 다른지 확인 가능
- sensitivity에서 어떤 파라미터가 가장 영향 큰지 식별 가능


AlphaFlow US에 백테스트 결과를 자동으로 분석하고 진단하는 리포트 시스템을 만들어라.

■ 배경
현재 백테스트 결과(daily_equity, trades)를 수동으로 해석하고 있다.
4-2(동적 청산), 4-1(ATR 사이징) 적용 후 반복 비교가 필요하므로 자동 리포트가 필요하다.

■ 할 일

1) api/services/backtest_reporter.py 신규 생성

   - asyncdefgenerate_report(backtest_id: str) -> dict
   - backtester._BACKTEST_RESULTS에서 결과 로드
   - 산출 지표:
     · 수익: total_return_pct, annualized_return_pct (252일 기준)
     · 리스크: MDD(%), MDD 지속일수, daily_volatility
     · 효율: Sharpe (rf=5% 연율), Sortino, Calmar
     · 거래: total_trades, win_rate, profit_factor, avg_win, avg_loss,
     avg_holding_days, max_consecutive_losses
     · 청산 분포: exit_reason별 건수 (trades에 exit_reason이 있으면 사용,
     없으면 return_pct로 추정: return_pct==-8.0 → "fixed_sl", ==15.0 → "fixed_tp", 나머지 → "other")
   - 벤치마크: yfinance로 SPY 동기간 수익률 조회, alpha = 전략수익 - SPY수익
     (yfinance 호출은 asyncio.to_thread로 감싸기)
   - 자동 진단 (diagnoses 리스트):
     · exit_reason에서 동적 청산 비율 < 5% → CRITICAL, "고정 SL/TP 종속"
     · 월 평균 거래 < 5건 → HIGH, "저빈도"
     · Sharpe < 0.5 → HIGH, "낮은 위험조정 수익"
     · alpha < -10 → CRITICAL, "심각한 벤치마크 언더퍼폼"
     · avg_holding_days > 25 → MEDIUM, "과도 보유"
   - severity 순 정렬 (CRITICAL → HIGH → MEDIUM → INFO)
   - asyncdefcompare_reports(backtest_ids: list[str]) -> dict
     · 여러 백테스트의 주요 지표를 나란히 비교 테이블로 반환
2) api/routers/backtest.py에 엔드포인트 추가

   - GET /api/backtest/results/{backtest_id}/report → generate_report 호출
   - POST /api/backtest/compare (body: {"ids": [...]}) → compare_reports 호출

■ 완료 기준

- 기존 Backtest 1, 2의 backtest_id로 리포트 생성 가능
- 리포트에 Sharpe, MDD, win_rate, exit_reason 분포, SPY alpha가 모두 포함
- 자동 진단 결과 3건 이상, severity 순 정렬
- compare에서 두 백테스트를 나란히 비교 가능

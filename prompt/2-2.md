당신은 퀀트/알고트레이딩 백테스트 시스템 전문 개발자입니다.

## 배경

AlphaFlow US의 `stock_daily` 테이블에 3년치 OHLCV + 기술지표 데이터가 저장되어 있습니다. 현재 시스템의 트레이딩 로직(스크리닝 → 스코어링 → 시그널 → 주문)을 과거 데이터에 대해 시뮬레이션하는 백테스트 엔진을 구축합니다.

## 현재 트레이딩 로직 요약

1. 스크리닝: numeric_score ≥ 60 또는 ≤ 35인 종목 필터
2. 스코어링: numeric_score (50%) + text_score (35%) + macro_score (15%) 가중합산
3. 조정: priced_in -15, RSI≥75 -10, RSI≤25 +10
4. 시그널: final_score ≥ 70 → BUY, ≤ 30 → SELL, else → HOLD
5. 포지션 관리: max_order_amount $1000, stop-loss -8%, take-profit +15%

## 작업 요구사항

### 1. `api/services/backtester.py` 생성

```python
Copyclass BacktestConfig:
    start_date: date
    end_date: date
    initial_capital: float = 100_000.0
    max_order_amount: float = 1_000.0
    max_positions: int = 20
    stop_loss_pct: float = -8.0
    take_profit_pct: float = 15.0
    screening_upper: float = 60.0
    screening_lower: float = 35.0
    buy_threshold: float = 70.0
    sell_threshold: float = 30.0
    w_numeric: float = 0.50
    w_text: float = 0.35
    w_macro: float = 0.15
    commission_pct: float = 0.0  # Alpaca 무수수료
    slippage_bps: float = 5.0    # 5bps 슬리피지 가정
    text_score_default: float = 50.0  # 뉴스 없을 때 기본값
    macro_score_default: float = 50.0
```

### 2. 백테스트 실행 플로우

날짜별 루프 (trading_days만):

```
for each trade_date:
    1. 해당 날짜까지의 기술지표로 numeric_score 계산 (look-ahead bias 방지)
    2. text_score는 text_score_default 사용 (Phase 3에서 뉴스 통합)
    3. macro_score는 macro_score_default 사용 (Phase 3에서 통합)
    4. 3축 가중 합산 + 조정 → final_score → BUY/SELL/HOLD
    5. BUY: 포지션 한도 미달 시 max_order_amount로 매수 (다음 날 시가 + slippage)
    6. SELL: 보유 종목이면 전량 매도 (다음 날 시가 + slippage)
    7. 보유 포지션 stop-loss/take-profit 체크 (당일 저가/고가 기준)
    8. 일별 포트폴리오 가치 기록
```

### 3. look-ahead bias 방지

* `numeric_score` 계산 시 `trade_date` 이전 데이터만 사용해야 합니다.
* 매수/매도는 시그널 발생일 다음 날 시가(open)로 체결합니다.
* stop-loss는 당일 저가(low)로 확인하지만 체결가는 stop-loss 가격으로 가정합니다.

### 4. 성과 기록

BacktestResult 클래스:

* daily_equity: list[tuple[date, float]] - 일별 포트폴리오 가치
* trades: list[dict] - 모든 체결 기록 (entry_date, exit_date, symbol, side, pnl, return_pct)
* signals: list[dict] - 모든 시그널 기록

### 5. API 엔드포인트

* `POST /api/backtest/run`: BacktestConfig를 받아 백테스트 실행, 결과 반환
* `GET /api/backtest/results/{backtest_id}`: 저장된 백테스트 결과 조회
